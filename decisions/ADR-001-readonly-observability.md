# ADR-001: MACS 只读不控制

**状态**：采纳  
**日期**：2026-07-14  
**决策者**：sg-architect  
**来源**：自发（MAEA Agent 自治原则）  

---

## 上下文

多 Agent 监控系统有两种范式：控制平面（可下发命令、重启 Agent、修改配置）和可观测平面（只读采集、展示、告警）。需要决定 MACS 的定位。

## 决策

MACS 是纯可观测层。只通过 A2A 协议轮询 Agent 健康端点，**永不下发命令、不重启 Agent、不修改配置**。

## 理由

1. **Agent 自治**：MAEA 的 Agent 是自主实体，MACS 不应成为集中控制者
2. **安全边界**：只读权限零攻击面，不需要 Agent 信任 MACS
3. **OPC 场景**：单人操作，Agent 挂了操作者手动重启比自动重启更安全
4. **实现简单**：无需 Agent 端安装 agent/plugin，纯外部观测

## 后果

| 影响 | 描述 |
|------|------|
| 正面 | 安全边界清晰，Agent 零侵入，实现复杂度低 |
| 负面 | 不能自动重启挂掉的 Agent，不能远程修改配置 |
| 风险 | 用户期望"一键修复"功能时需要解释为什么不做 |

## 备选方案

| 方案 | 评估 | 原因 |
|------|:----:|------|
| A: 控制平面（可下发命令） | ❌ | 需要 Agent 信任 MACS，增加攻击面，违背 Agent 自治原则 |
| B: 混合（只读+手动触发重启） | ❌ | "手动触发"本质就是控制，边界模糊比明确不做好 |
