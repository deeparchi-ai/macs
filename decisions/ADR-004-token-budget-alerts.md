# ADR-004: Token 预算两级告警

**状态**：采纳  
**日期**：2026-07-14  
**决策者**：sg-architect  
**来源**：MAEA 框架（WLM Token 预算）  

---

## 上下文

MAEA 的 WLM 定义了 Agent 的 Token 预算体系。MACS 需要可视化并告警预算状态。需要决定告警阈值和动作。

## 决策

两级告警：80% 月度预算 → 软告警（DM 通知操作者）；100% → 硬阻断（Agent A2A 返回 503）。

## 理由

1. **提前预警**：80% 时操作者有 20% 余量做决策（暂停非关键任务、手动加预算、或接受超支）
2. **不自动续费**：MACS 不替操作者做财务决策
3. **可覆盖**：操作者可以在 dashboard 手动提升预算上限

## 后果

| 影响 | 描述 |
|------|------|
| 正面 | 防止 Token 账单失控，给操作者缓冲时间 |
| 负面 | 100% 硬阻断可能导致 Agent 任务中断 |
| 风险 | 如果 Agent 在关键任务中被阻断，需要快速覆盖机制 |

## 备选方案

| 方案 | 评估 | 原因 |
|------|:----:|------|
| A: 仅告警不阻断 | ❌ | Token 超支是财务风险，必须有强制机制 |
| B: 三阶段（70/85/100） | ❌ | 两阶段足够，三阶段增加复杂度但不增加价值 |
